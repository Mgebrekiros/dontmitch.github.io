allow me to introduce myself

Who I am

My name is Mitch, and I'm a mechanical engineer turned full stack web developer. You can read a bit more about me on my homepage, but here's the abridged version:

I love learning. More than anything else, that's what I'm passionate about. I am constantly trying to expand on my abilities or develop new ones. I enjoy reading - especially about tech news - and tend to not watch TV - excluding sports - for the sole reason that I'd rather be learning new things. I wholeheartedly embrace my inner engineer and can geek out with the best of them.

Now that I've graduated from Dev Bootcamp's first Chicago cohort, I'd love nothing more than to continue my learning at a quality company, preferably 37signals.

What I bring

If there's one thing I know, it's that 37signals is on the top of most people's dream-job list. People don't do this for just any company... So why is it that I think I can be considered for a position there?

First and foremost, although I'm new to the industry, I have been formally trained as an engineer. I graduated from Northwestern University at the top of my class. At face value, that means nothing more than that I was good at school, but I choose to interpret it as meaning that I have the mindset of an engineer and a killer work ethic.

Second, I am great at knowing when I don't know something, admitting to that fact, and seeking out help. In short, I thrive on the greatness of others. This is coincidentally the reason I want to work at 37signals so badly - the opportunity to join the 37signals team is what draws me there above all else.

Third, I learn quickly. I encourage you to continue reading with this fact in mind: I have been formally trained as a web developer for just nine weeks. I have another three weeks of formal training ahead of me and fully except to continue learning at the same pace that I learned at these past nine weeks.

What I've done

You can see all of my work on my GitHub page, but here are three projects that I'd like to highlight:

grouPACT

This was the longest project that I've worked on to date, spanning eight days with two team members. It's a Rails-backed web application featuring some advanced algorithms run as background jobs, mailer integration and Stripe authentication and payment processing. We made heavy use of GitHub for version control (we had well over 500 commits and 150 feature branches at the end), Asana for group planning and task management, and RSpec and Capybara testing for quality control.

You can find the live site at grouPACT.me and can view the full source code here.

The code is by no means perfect but is hopefully an indication of how far I've come over the past few weeks. As a sidenote, I learned Rails a week and half before starting this project.

Flash Cards

This is a Sinatra-backed flashcard application that I built after two weeks of JavaScript, jQuery, CSS and HTML instruction. The project took me the better part of a weekend and was a great learning experience. You can see a working version on heroku here and the source code here. Some of the basic features include: secure password handling using BCrypt, redirect backs using session cookies, AJAX calls, advanced CSS animations, and custom layouts (no bootstrap here). I purposefully haven't cleaned up the code to allow for an accurate gauge of my skill level at that point in time.

Intro to Rails

Shortly after being introduced to Rails myself, I decided to capture my onboarding experience while it was still fresh in my mind by writing my own beginner's guide to Rails. My hope is that future boots will find it useful leading up to their transition from Sinatra to Rails. The readme should explain how to use the guide, but it is worth noting that this is still largely a work in progress.
